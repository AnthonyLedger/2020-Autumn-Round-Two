什么是HTML?

正经的话不想讲, 因此被删除了

简单且随意地说, html主要表明了一个网页中包含了什么东西..

一段html代码可能会像这样:

<html>
    <head></head>
    <body>
        <div id="father" class="box" contenteditable>
            <!-- comment in html! -->
            <div id="child1" class="box boxes">???</div>
            &lt;Some code here..&gt;<br />
            <span id="child2">
                And Some&nbsp; other code here.. &#708;_&#708;
            </span>
        </div>
    </body>
</html>

它(至少在chrome上)显示出来大概会长这样:

那么这段代码中包含了哪些html的知识点呢?

  • 由一对尖括号括起来的, 形如<div>这种的, 叫html标签(tag). 标签可能会由如下几个部分构成:

    • 标签的最开始, 如div啊这种, 叫标签名称(tag name). 不同的标签名称标志了不同的元素, 除了上面出现的div``span, 还有其他比如img之类的, 请大家自行了解ww
    • 形如id="child1"或者class="box"这样的东西, 被称为属性(attribute), 它们位于元素前面的标签中, 说明了这个元素的一些特性. 属性分为两部分:
      • 像id class这样的是属性名称(attr-name)
      • 像child1 box这样的是属性值(attr-val)
      • 属性名称是不能省略的, 而属性值是可以省略的.. 至于省略属性值之后会发生什么, 欢迎大家尝试一下~
      • 有一些特殊的属性:
        • id: 这个就是表示了元素的id, 作为一个合格的id, 它应该是唯一的.
        • class: 这个表示了元素的类, 通过把多个元素设置成相同的class, 可以方便你对它们同时进行一些相同的操作..
        • 其他的属性请大家自行查找资料学习ww

  • 形如<div id="child1" class="box">???</div>这样的东西, 被叫做元素(element).

    • 那些不在标签里的, 它们则表示了元素的内容, 大概就是你看到的那些字.
    • 如同C中代码块应该被一对{}包围一样, 每个html元素也应该是由一对标签包围, 这一对标签的名称应该是相同的, 不过后面的要在前面加一个/.
      比如<div></div>这样.
    • 元素间可以嵌套, 就像大括号或者语句一样. 这表示元素之间有包含关系, 是的, 我们常用父子啊之类的这种辈分关系来说明它们的包含关系..

  • <!--xxx-->这个是html中的注释, 无论你之前学过任何什么编程语言, 你应该都知道注释. 就是说这个东西它只起到说明 注解的作用, 而不表示html中的任何元素.

  • 当我们真正需要尖括号之类的东西的时候, 可能会导致html代码出现混乱, 因此我们需要转义字符. 比如用&lt;代替<, &gt;代替>之类的..
    其他转义字符请大家自行了解ww

  • 一个成熟的html界面应该学会由<html></html>这样的标签包裹

    • 嗯当然现在的界面通常最开始是这么一行: <!DOCTYPE html>, 这是个啥大家可以自行查一下ww

  • <head></head>这一部分呢, 是用于定义文档的头部，其中的元素可以表示引用js啊, 标记去哪里找css啊等等..一般来说绝大多数head的元素都不是对用户显示的.. 有啥元素啥的自己了解就好ww 有一个挺重要的提一下就<title>xxx</title>这个是用来说明网页标题的(就浏览器标签页上写的那玩意..)

  • <body></body>这一部分呢, 是用于定义文档的主体, 一般对用户可见的内容都会在这里呈现.. body可以有属性, 但是已经不被赞成使用, 请尽可能使用css来完成它们的效果~

什么是CSS?

正经的话不想讲, 因此被删除了

简单且随意地说, css主要表明了一个东西应该长什么样子..

一段css代码可能会是这样:

#father .box[id^="c"] {
    border: black 2px solid;
    background-color: red;
    color: 15;
    font-weight: 2px; 
}
.box:before {
    content: "Hello: ";
}
span {
    color: aqua;
}

这样和上面的代码采取合适的方式配合之后, 效果是这样的:

css如何和html结合呢?

  1. 一种方式是通过style属性, 这样会作用于当前的元素. 比如这样:
    <div style="font-size:14px;color: red;">555</div>
  2. 一种是通过style标签(通常是一个放到head部分的元素, 来影响整个界面), 比如这样
    <head>
    <style>
        body {
            text-align: center;
        }
    </style>
</head>
  3. 一种是通过引入外部文件的形式, (这个一般也是放到head里面, 造成全局影响啊~)
    (当然我们这次的任务希望你提交一个单独的html文件, 因此可以不用引入外部css文件啦~)
    <link rel="stylesheet" type="text/css" href="css/style.css"/>

一段合格的css代码又包含了哪些东西呢?

  • css中包含了各种规则(rule), 一条规则形如color: aqua;, 是一个典型的key-value对.

    • 前面key表明要对什么进行设置
    • 后面value表明要设置成什么
    • 有一些key可能是很多key的简写, 这样可以给它赋很多的value,
      例如margin(这个表示外边距, 后面应该会有讲到?), 会包括:
      • margin-top
      • margin-right
      • margin-down
      • margin-top

    之类的, 因此margin: 8px 4px 2px 1px, 表示当前这个元素上外边距8px 右外边距4px 下外边距2px 左外边距1px.. 那么如果我们缺省了一些值的话呢? 这个还请大家自己尝试一下啦ww

  • css中出现的px表示pixel(像素), 当然啦, css中不只有这一种单位哦, 其他的单位以及相关的一些知识请大家自行了解啦ww

  • 为了说明一套规则要对哪些元素适用, 就需要用到选择器(selector), 其实就是放在{}前面的那写东西 选择器有很多种:

    1. 可以直接检索标签名的, 比如div{}, 将对所有<div>元素生效.
    2. 可以检索id的, 比如#father{}, 将对所有id="father"的元素生效 (当然, 我们总是希望id是唯一的, 这样的元素只有一个).
    3. 可以检索class的, 比如.box{}, 将对所有class="box"的元素生效.
    4. 可以使用正则表达式(不知道什么是正则表达式的请自行查询ww), 比如a[id^="b"]将检索所有id以b开头的<a>元素.
      其他用法欢迎大家自行查询并且了解ww
    5. 一些别的选择器用法(比如什么:before)请大家自行了解ww(大概在这次的任务中完全用不到呢)

  • 形如selector{rule-key: rule-value;...}这样的被称为ruleset, 还有一种以@开头的at-rules, *感兴趣的请大家自行了解叭ww. 这些ruleset和at-rules(at-rules可以包含ruleset哦~) 就构成了一套完整的CSS呢~

  • at-rules也有不少作用, 比如如果想从本地文件中导入字体的话, 不妨使用@font-face啦, 它的用法请大家自行学习了解啦ww

知道了这些, 再加上亿些额外的了解, 应该足以帮助你完成我们的任务.

稍微讲点布局?

在进行网页设计和布局的时候, 我们常常把每一个html元素看成一个盒子.

在CSS中, 我们会使用一个概念, 盒模型(box model).

它本质上来说, 是一个封装HTML元素的盒子，包括了以下几个部分:

  1. 外边距 margin
  2. 边框 border
  3. 内边距 padding
  4. 内容 content

其中, padding和content两部分内容, 组成了背景(background).

box们有两种常见的展示方式, block和inline, 你可以通过display来选择展示方式.

  • block们非常霸道, 要自己独占一行的空间.
  • 而inline们会尽可能往后排, 只有需要的时候才会换行.

在没有指定宽和高的情况下, 关于box的width和height属性, 有

width height
block max min
inline min min

其中,

  • min表示足够盛下所有内容的最小值
  • max表示和它的父元素的值一样大小

那如果内容大到盛不下了呢?

  • 对于超过宽度的情况, box会主动把自己垂直拉长以适应内容
  • 对于超过高度的情况, box会让内容溢出
  • 当然对于这里行为的控制, 可以使用overflow进行控制

对于我们之前讨论过的所有boxes, 他们在css中都有一个position来控制它们的位置. 在你没有对它进行设置之前, 它都保持默认值static.

position还有一些其他的取值, 比如(一些你应该了解而且用得到的):

  • fixed
  • relative
  • absolute

对这几个取值的理解, 可能是在初学css的时候最难的部分了吧..

尤其是relative和absolute的区别, 可能需要自己多加体会ww
